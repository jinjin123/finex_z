1、接口数据规范

说明：返回值200为成功，其他为异常

范例：
接口负责人：XXX							
请求方式：POST
请求参数：Controller/method       // 控制器/方法
POST到服务器的json数据：
{
	"type":"1",          // 字段说明
	"orderId":"123"      // 字段说明
}
成功返回：
{
	"code": 200,
	"msg": "",
	"data": [ {
            "order_id": "1",
            "bank_num": "123456789987564321",
	        } ]
}
或者返回空数组：
{
    "data": [],
    "code": "200",
    "msg": "success"
}

错误数据
失败返回：
{
    "data": [],
    "code": "201",                 // 错误代号
    "msg": "错误信息"                 // 错误信息
}

2、控制器、模块、common/function.php说明

说明：

  • 1、参数判断都在控制器内做判断，业务逻辑代码基本不在控制器类写
  • 2、业务逻辑代码基本写在逻辑类或模块类里面，基本不再做参数入库的判断
  • 3、在控制器内做事务处理，不在模块类或逻辑类做，防止事务嵌套
  • 4、Common/function.php,这个文件基本不要有数据的读写，部分特殊的例外，如post_log这里的日志
  • 5、写funtion时要初略已有的fun函数和框架提供的函数，不要重复造轮子，fun命名要有意义的英文单词或缩写，不能中英文结合
  • 6、fun，类和方法，要写注释和作者
